# ChurchCMS Web Application & API

The ChurchCMS web application is built with **Laravel 8+**, **Vue 2**, and **Laravel Mix**. It provides both a comprehensive admin interface and a REST API for integrating with external applications and mobile apps.

## Installation Methods

ChurchCMS offers two flexible installation approaches to suit your needs:

### 🌐 Browser-Based Visual Installer (Recommended for First-Time Users)

A step-by-step web interface that guides you through the entire setup process.

**Best for:**
- Non-technical users
- Local desktop setup
- Visual feedback and progress tracking
- Users new to ChurchCMS

**What it covers:**
1. Welcome & environment check
2. System requirements verification
3. Database configuration
4. Church information setup
5. Admin user creation
6. Automated installation progress
7. Completion confirmation

**Getting started:**
1. Clone the [Laravel repository](https://github.com/church-cms/church-cms-laravel)
2. Configure `.env` with basic database details
3. Access the installer at `/installer` in your browser
4. Follow the visual wizard (6-7 steps)

**Installation time:** ~5-10 minutes

---

### 💻 CLI Installer (Recommended for Developers)

A command-line installation method perfect for automation, servers, and developers.

**Best for:**
- Developers and power users
- Automation and CI/CD pipelines
- SSH/VPS server setup
- Scripted deployments
- Headless server environments

**Quick start:**
```bash
# 1. Clone repository
git clone https://github.com/church-cms/church-cms-laravel
cd church-cms-laravel

# 2. Configure environment
cp .env.example .env
nano .env  # Update database credentials

# 3. Run framework setup
bash install.sh

# 4. Create church and admin
php artisan church:setup

# 5. Start development server
php artisan serve

# 6. Access at http://localhost:8000/admin
```

**Installation time:** ~2-3 minutes

## System Requirements

Both installation methods require:

**Server:**
- PHP 8.2 or higher
- MySQL 5.7 or higher
- 256MB minimum RAM (512MB recommended)
- Composer (auto-installed during setup if available)
- npm/Node.js (auto-installed during setup if available)

**Required PHP Extensions:**
- PDO & PDO MySQL
- Mbstring, OpenSSL, Fileinfo
- Tokenizer, XML, Ctype, JSON

**Optional Services** (for full functionality):
- S3-compatible storage (AWS S3, DigitalOcean Spaces, etc.)
- Firebase Cloud Messaging (push notifications)
- Twilio (SMS communications)
- Pusher (real-time features)
- Analytics integrations

## Common Questions

**Q: Which installation method should I use?**
- Use the **visual installer** for your first time or if you're not familiar with terminal commands
- Use the **CLI method** if you're a developer or need to automate setup

**Q: Can I switch between installations?**
- Yes, both methods result in the same working system
- You can even reinstall with the other method later

**Q: What if installation fails?**
- Check the error logs in `/storage/logs/laravel.log`
- Verify all system requirements are met
- Ensure database credentials are correct
- Contact support with error messages and logs

**Q: Do I need to know Laravel/Vue to use it?**
- No, the web interface requires no coding knowledge
- Use it as a complete admin dashboard
- Only developers need Laravel/Vue knowledge for customization

**Q: Is there a demo available?**
- Visit [churchcms.app](https://churchcms.app) for demo information
- Contact the community for test access
